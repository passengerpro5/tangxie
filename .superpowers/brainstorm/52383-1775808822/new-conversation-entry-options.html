“新建会话”入口方案

目标：在“安排任务”页里随时可新开会话，不打断输入主链路，也不挤压“安排”发送动作。

A. 标题行右侧文字按钮
安排任务
历史记录
+ 新会话
聊天内容区域
补充 deadline、时长或开始时间

方案 A

入口放在顶部 tab 行右侧，最符合“当前会话级操作”的语义，也不会和附件加号混淆。

优点

  • 看起来像“切换/管理会话”的一部分
  • 不会误解成“上传附件”或“发送”
  • 视觉权重可控，容易做轻

缺点

  • 需要重新整理顶部横向空间
  • 如果文案过长，窄屏会偏挤
B. 对话区顶部胶囊按钮
安排任务
历史记录
+ 新建会话
聊天内容区域
补充 deadline、时长或开始时间

方案 B

入口放在线程区域顶部，和消息内容同层，用户进来后第一眼就能看到。

优点

  • 很显眼，发现成本最低
  • 不影响顶部 tab 的现有节奏

缺点

  • 会侵占首屏消息空间
  • 像“会话内容的一部分”，语义没那么干净
C. 输入框右上角小图标
安排任务
历史记录
聊天内容区域
新会话
补充 deadline、时长或开始时间

方案 C

入口贴近输入区，强调“换个话题重新开始”。

优点

  • 操作路径短
  • 和发送动作同区域，临场切换快

缺点

  • 容易和附件加号、发送按钮形成竞争
  • 输入区已经比较满，再加入口会更挤

推荐

我推荐 方案 A。它最符合“当前会话管理”的语义，位置稳定，不会破坏你已经打磨过的输入区，也最不容易和附件入口产生歧义。
